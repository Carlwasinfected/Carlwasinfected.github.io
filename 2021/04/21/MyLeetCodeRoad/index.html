01.01. 判定字符是否唯一

难度：简单

notes：

这题考查去重。 显然最直接的解法是通过set或者dict判断其中有已存在的char即可。

但这题还可以通过位运算来解决。「用位数组代替set 解决上面数据结构带来的空间开销较大的问题」

即记录一个值为0的int型变量，取后26位作为记录的bool array 那么每一位其实就代表了这个字符与a之间的距离。如果这一位为零，那么是第一次见，将之置一「之所以这么做是因为下标距离与字符满足1:1的映射关系」;下次访问时若该位为一，则代表已经有此字符，返回False

PS: 1 << len_to_char_a是将0000 0001左移char - 'a'个比特的距离

notes：

这题的思路很直接。首先如果字符串的长度不等，那肯定不能重排。其次，为了达到重排，两个string的各个字母出现的次数应该是一致的。那么直接用两个字典<char, int>记录下两个字符串的字母出现次数，再遍历比较下各个value是否一致就可以了。

这题思路总体和3sum类似的，只不过while的时候需要计数一下重复元素有多少个，然后分两种情况进行排列组合「1.left->right之间都相等. 那就是C(r-l+1)2 2. 不等.」

有以下几个坑:

  1. 对于左右不等的情况，不要用arr[left] == lval判断，这样会使得退出左指针的 while 的时候，左指针已经指向了下一个非重复元素. 可能由此导致少解.

    e.g., 对于target == 7 arr[2,2,3,3,4,4,5,5]来说，在做[3,3,4,4]判断时，left从arr[2]–>arr[4]结束，故right从arr[5]向左走的时候，走一步就会触发left == right退出循环，因此第一个[4]的组合就没有算进去.

    此时若print(leftnum, rightnum, arr[left], arr[right]) ，结果为2 1 4 4 显然rightnum少算了一个. 这是由于 left 指针提前到了arr[4] :4这步而产生的Bug.

    本题我们要精确计数当可解时，左、右各有重复数字多少个. 因此在退出 while 时 我们要保证左、右指针各在自己搜索方向的最后一个重复元素上，而不是多走了一步，已经指向下个不重复的元素.

    而labuladong的模版之所以是while left < right and arr[left] == lval 是因为书中的题目只要找到一个解即可，而不像本题，必须关注所有重复解的个数. 因此while left < right and arr[left] == lval可以保证在退出两个 while 时， 左右指针已经都多走了一步.

    如果为了保险，可以都选择采用下面代码中的“先走到最后一个重复元素，两个循环结束后再更新下一步”的做法

    1. ans比较大，更新后马上取模

滑动窗口

Notes

滑动窗口的窗口上下限是 [left, right )，它的区间是左闭右开的.也就是说除了取出arr[right]后马上有right++ ，所以在其他任何时刻arr[right]均不在窗口内. 因此滑动窗口的长度为right - left，而不是常见的right - left + 1

字符串题，格外注意valid == need.size()的条件，基本会在某个if中用到. e.g., 收缩条件； 结束条件

When 滑窗：子序列 子串 区间内乘积 区间内加和 最大最小k XOR

滑窗特点

  1. 每次迭代，left不一定+1， 但right一定会每次有+1
  2. 收缩的前提时，当前[left, right)已经包含了正确解，但需要优化掉其余不合题意的解（子集超集关系）
  3. 窗口长度是right - left
  4. 复杂度是线性的 O(n)

1584. Min Cost to Connect All Points｜最小生成树

生成树采用Kruskal算法；同时由于需要判断两个节点是否已经连通，所以还需要用到并查集。

Kruskal（克鲁斯卡尔）算法的基本思想如下：

①对所有给的边按其边权进行从小到大的排序。

②顺序遍历排序后的边。

③若当前边连接的两个节点已经在生成树上了「通过UF」，则跳过这条边。

④否则则将这条边加入生成树。

⑤当连通分量为1时，所有节点已连接；结束算法，此时生成树就是最小生成树。

PS:

  1. cost数组写为[[i, j, cost] ... ]的三元组比较方便. 可直接根据cost值排序，然后遍历取i, j即可
  2. 自定义sort和sorted的排序方式有两种办法.
    1. 自定义新class, 改写__lt__
    2. 写一个函数，函数名作为参数传给排序函数 lst.sord(key=mykeyfunc) ... def mykeyfun(x): ...
